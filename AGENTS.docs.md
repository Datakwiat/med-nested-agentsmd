## Trigger
- Run this file when rearchitecture indicators are detected.

## Checks
- New or changed files in infra or config directories.
- API, route, interface, or contract changes.
- Environment variable or dependency changes.
- New workflow, pipeline, or process.

## Action
- Update changed module README and parent README.
- Add concise "Recent Changes" summary.
- Update architecture diagrams; for major features, add system and transaction diagrams.

## Discovery
- Use `.docs/readme-discovery.md` only when README scope is unclear.
